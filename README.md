# Уязвимость Path Traversal  

Данный репозиторий содержит YAML-шаблон Nuclei для детектирования уязвимости Path
Traversal с использованием Nuclei Code Protocol (Python).    

## Описание уязвимости  
Path Traversal представляет собой уязвимость, которая позволяет злоумышленнику получить доступ к файлам и директориям, находящимся за пределами предполагаемой корневой директории веб-сервера.  
Эксплуатация достигается путем манипулирования переменными, которые ссылаются на файлы (например, через URL-запросы), используя специальные последовательности символов, такие как "../" (возвращающиеся на уровень выше в структуре каталогов).   

**Severity.** Уровень критичности можно оценить в зависимости от того, какие именно права доступа есть у пользователя. Если пользователь имеет права root, то он может осуществить RCE на сервере или полностью захватить систему, что соответствует критическому или высокому severity (High/Critical). Если пользователь не имеет прав на изменение или создание файлов и каталогов, но при этом имеет доступ к конфиденциальным данным, то severity может быть от среднего к высокому (Medium/High).  

**Impact.** Нарушение целостности и конфиденциальности данных: чтение файлов или каталогов (системные файлы, API-ключи, конфигурационные файлы, переменные окружения, SSH-ключи); изменение файлов или каталогов; обход механизмов защиты.  
 
**Recommendations:**    
Для того чтобы избежать уязвимости Path Traversal, необходимо придерживаться следующих принципов secure coding:  
1) Строгая проверка вводимых пользователем данных: использования белых списков с разрешенными типами и названиями файлов.  
2) Приведение введенного файлового пути к каноническому (абсолютному) виду: разрешение последовательностей символов для обхода каталога (../, .%00./, %2e%2e%2f и т.д. ) и проверка на то, принадлежит ли данный файл к разрешенному каталогу.  
3) Принцип наименьших привилегий: у учетной записи пользователя веб-приложения должны быть только те права доступа, которые необходимы для выполнения его задач.  


## Инструкция по использования  

### 1. Развёртывание уязвимого сервиса

Убедиться, что установлены Docker и Docker Compose.  

Создать файл `docker-compose.yml`:  

```yaml  
version: '3'  
services:  
  path_traversal:  
    image: projectasuras/path_traversal  
    ports:  
      - "5001:5000"  
    container_name: vuln_app  
```

Запустить контейнер:  

```bash  
docker compose up -d  
```  

Проверить запуск:  

```bash  
docker ps  
```

Сервис должен быть доступен по адресу:  

```text  
http://127.0.0.1:5001  
```  

##№ 2. Подготовка к использованию шаблона  

Шаблон использует Code Protocol на Python и требует специальной версии Nuclei с поддержкой `python3`.   
Для безопасного выполнения шаблон необходимо подписать.  

1. Создайте ключ подписи (один раз)  

```bash  
nuclei -sign  
```   
2. Подпишите шаблон  

```bash  
nuclei -sign -t path-traversal-detection.yaml  
```  

### 3. Проверка уязвимости

Запустить детектирование:

```bash
nuclei -u http://127.0.0.1:5001 \
-t path-traversal-detection.yaml \
-code \
-esc
```
